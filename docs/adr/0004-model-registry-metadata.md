# Populate `Base.metadata` through a central model registry

`src/registry.py` imports every ORM model module purely for its side effects, so that importing this one module guarantees `Base.metadata` is fully populated. Code paths that need the whole schema — the FastAPI app, Alembic migrations, and the seed script — import `registry` (not the individual model modules) before relying on the mappers.

We chose this because the application's routers only reach a subset of model modules, but SQLAlchemy needs every mapped class loaded before it can configure mappers and resolve cross-domain foreign keys (for example `refresh_tokens -> users`, where the app never routes the users module). Without a single import point, whether the schema is complete depends on which modules happened to be imported first — a fragile, order-dependent failure that surfaces as confusing "failed to locate a name" mapper errors, and that Alembic autogenerate hits hardest because it imports the metadata directly. Centralizing the imports makes completeness explicit and gives migrations and seeds one reliable thing to import. The trade-off is a module of "unused" imports that linters flag (hence the `# noqa: F401`), and a rule the team must remember: a newly added model module has to be registered here or it silently drops out of the schema.
